# AGENTS.md — notrack-link

Cloudflare Worker that follows redirect chains and strips tracking parameters
using ClearURLs rules. React 19 SSR + CSR frontend styled with Radix UI Themes.

## Tech Stack

- **Runtime:** Cloudflare Workers (compatibility date `2024-09-25`, `nodejs_compat`)
- **Language:** TypeScript (strict mode) + TSX (React 19, automatic JSX transform)
- **Package manager:** pnpm (v10)
- **Node version:** LTS/Jod (22.x), managed via `mise.toml`
- **Frontend bundler:** Vite (entry: `src/csr.tsx` → `build/build.js` + `build/build.css`)
- **Worker bundler/deployer:** Wrangler
- **UI library:** Radix UI Themes (`@radix-ui/themes`)
- **Test framework:** Vitest 2.0 with `@cloudflare/vitest-pool-workers`
- **Formatting:** Prettier (no ESLint)

## Commands

```bash
pnpm install                              # install dependencies
pnpm dev                                  # local dev server (wrangler dev)
pnpm fe-build                             # clean build of frontend assets (vite build)
pnpm fe-dev                               # vite watch mode
pnpm deploy                               # fe-build + wrangler deploy
pnpm test                                 # run all tests
pnpm vitest run test/index.spec.ts        # run a single test file
pnpm vitest run -t "unit style"           # run tests matching a name pattern
pnpm tsc --noEmit                         # type-check only (no emit)
pnpm cf-typegen                           # regenerate worker-configuration.d.ts
```

No `format` or `lint` script exists — run Prettier manually if needed.
Tests run inside the actual Workers runtime via `@cloudflare/vitest-pool-workers`.

## Project Structure

```
src/
  index.ts      — Worker entry: fetch handler, UnwrapAPI (cross-worker RPC)
  scout.ts      — Link-following engine, ClearURLs rule processing
  fe.tsx        — React App component (all UI)
  csr.tsx       — Client-side hydration entry (esbuild entry point)
  ssr.tsx       — Server-side rendering (renderToReadableStream)
  runtime.ts    — isBrowser() / isCloudflare() env detection
test/
  index.spec.ts — Vitest integration tests (.spec.ts suffix)
  tsconfig.json — Test TS config (extends root, adds workers types)
  radix-stub.ts — @radix-ui/themes stub for the Workers test runtime

worker-configuration.d.ts  — generated by wrangler types (do not edit)
```

Flat structure — no nested directories in `src/` or `test/`.

## Architecture

- **SSR + hydration:** Worker renders HTML via `ssr.tsx` (`renderToReadableStream`),
  browser hydrates via `csr.tsx` (`hydrateRoot`). Both wrap `<App>` in Radix `<Theme>`.
  Vite config (`vite.config.ts`) uses `rollupOptions` with a fixed `[name].js`/`[name][extname]`
  output pattern so asset filenames stay predictable (`build.js`, `build.css`) without a manifest.
- **SSE streaming:** `/iterhops/<url>` streams hop-by-hop results as Server-Sent Events.
  A `TransformStream` + `queueMicrotask` pattern lets the `Response` return immediately
  while the async hop loop writes to the writable side.
- **Asset versioning:** `env.CF_VERSION_METADATA.tag || env.CF_VERSION_METADATA.id` is
  injected as a URL segment (`/assets/<version>/build.js`). The Worker strips the version
  prefix before proxying to `BUILD_ASSETS: Fetcher` and caches results in `caches.default`.
- **ClearURLs rules:** Fetched from a remote URL and cached in both the Workers runtime
  cache and `caches.default`. Tracking params are stripped at the final hop.
- **Cross-worker RPC:** `UnwrapAPI extends WorkerEntrypoint` exposes a `follow()` method
  for service bindings from other Workers.
- **Radix UI test stub:** `test/radix-stub.ts` is aliased to `@radix-ui/themes` in
  `vitest.config.mts` because Radix depends on `react-remove-scroll` which uses browser
  DOM APIs unavailable in the Workers test runtime.

## Code Style

### Formatting (Prettier)

- **Print width:** 140 — lines can be long
- **Quotes:** Single quotes for JS strings; double quotes in JSX attributes (Prettier default)
- **Semicolons:** Always
- **Indentation:** Tabs (not spaces)
- **Trailing commas:** Prettier 3 default (`"all"`) — present in multiline structures
- **Line endings:** LF

### Imports

Order within a file (no blank lines between groups, all contiguous):

1. Side-effect imports (`import '@radix-ui/themes/styles.css'`)
2. Platform/runtime imports (`cloudflare:workers`, `node:path`)
3. Third-party libraries (`@radix-ui/themes`, `react`, `react-dom/server`)
4. Local/relative imports (`./scout`, `./ssr`, `./runtime`)

The automatic JSX transform (`react-jsx`) means React does **not** need to be imported
in TSX files — the sole exception is `csr.tsx` which explicitly imports `React` because
it also calls `hydrateRoot`.

### Naming Conventions

| Entity                 | Convention       | Example                           |
|------------------------|------------------|-----------------------------------|
| Functions (top-level)  | camelCase        | `followLink`, `cleanLink`         |
| React components       | PascalCase       | `App`                             |
| Classes                | PascalCase       | `UnwrapAPI`                       |
| Exported types         | PascalCase       | `LinkHop`, `ValidURLString`       |
| Internal/private types | PascalCase       | (lowercase only for truly private module-local aliases) |
| Interfaces             | PascalCase       | `RulesProcessor`                  |
| String/numeric consts  | UPPER_SNAKE_CASE | `RULES_JSON_URL`, `MAX_REDIRECTS` |
| Object/mutable vars    | camelCase        | `rulesRuntimeCache`, `currentURL` |
| Data/API property keys | snake_case       | `status_code`, `removed_params`   |
| Files                  | lowercase, short | `scout.ts`, `fe.tsx`, `ssr.tsx`   |
| Test files             | `.spec.ts`       | `index.spec.ts`                   |

### Types vs Interfaces

Strongly prefer `type` for data shapes, unions, and aliases. Use `interface` only for
behavioral contracts with methods (e.g., `RulesProcessor` with a `match` method).

### Type Patterns

- **Discriminated unions** on a boolean literal: `LinkHop` is `{ final: false; … } | { final: true; … }`
- **Branded types:** `ValidURLString = string & { __valid_url: string }` with an
  `isValidURLString(s): s is ValidURLString` type guard
- **`satisfies` keyword:** Used on the Workers default export (`satisfies ExportedHandler<Env>`)
  and on yield expressions in `followLink` (`satisfies LinkHop`) to verify shape without widening
- **`(e as any).message`** in catch blocks — `useUnknownInCatchVariables` is not enabled
- **Workaround casts:** `Array` cast to describe `Array.fromAsync` which has no TS lib type yet

### Function Style

- `function` keyword for all named, top-level, and React component functions
- Arrow functions only for callbacks, inline event handlers, and object method values
- React components always use `function` declarations, never `const Component = () =>`

### Exports

- Named exports everywhere: `export function`, `export type`, `export class`
- **One** default export: the Worker handler object in `index.ts` only
- No barrel re-export files

### Error Handling

- Errors sent over SSE as `hop_error` events: `{ message: string }` JSON payload
- `.catch()` on fetch promises returns a typed `FetchFailure` object rather than throwing,
  enabling `Promise.race` between fetch and a timeout
- `null` return from functions like `parseRegexp` for non-fatal parse failures (fail-open)
- Empty `catch {}` for intentional silent fail paths
- `try/catch` in async SSE writer; errors written to the stream, then the writer is closed
  in `finally`

### Comments

- Sparse — no JSDoc, no section headers
- Short inline `//` comments only where behavior is non-obvious
- `console.log` / `console.time` for cache tracing and observability
- Stub files and workarounds get a brief explanatory comment block

### TypeScript Configuration

- `strict: true`, `noEmit: true`, `isolatedModules: true`
- `target: es2021`, `module: es2022`, `moduleResolution: Bundler`
- `jsx: react-jsx` (no React import needed in TSX)
- `types: ["@cloudflare/workers-types/2023-07-01"]` in root; test dir adds
  `@cloudflare/workers-types/experimental` and `@cloudflare/vitest-pool-workers`
- Root `tsconfig.json` excludes `test/`; `test/tsconfig.json` extends root and re-includes it

## Cloudflare Workers Patterns

- `Env` is a global interface generated by `wrangler types` into `worker-configuration.d.ts` —
  never import it, just reference it by name
- `ctx.waitUntil(...)` for fire-and-forget cache writes after the response is returned
- `caches.default.match` / `caches.default.put` for HTTP-level response caching
- `queueMicrotask(async () => { … })` to detach an async streaming loop from the main handler
- `HTMLRewriter` used inline with an anonymous class: `new (class implements HTMLRewriterElementContentHandlers { … })()`
- Asset URL pattern: `/assets/<CF_VERSION_METADATA>/filename` — version stripped before
  proxying to `BUILD_ASSETS.fetch()`
